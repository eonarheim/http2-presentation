HTTP/2

HTTP/2 is the new version of HyperText Transport Protocol(HTTP), which was released as an IETF standard in early 2015. (scary gov't agency joke IETF)

What's new in HTTP/2

  • Built in connection multiplexing
  • HPACK header compression
  • Server Push
  • Request prioritization
  • Binary in transit
  • HTTP/2 is SSL only***
  • Backwards compatability to HTTP/1.1 with protocol negotiation
  • The HTTP/1.1 verb semantics you know and love are the same

What does multiplexing get me?

  • Bundling might not be the best idea
  • Inlining/Bundling/Concatenating/Spriting limits the effectiveness of your cache
  • Ahead of line connection blocking is gone
  • Fewer longer lived connections

Header Compression w/HPACK! Egaaads!

Uses a technique called huffman encoding to transform header-value pairs into a compact numeric representation.

Awesome if you have a lot of cookies and other information crammed in your header

Static table - "known" header field and common values each with its own index number that replace text in transmission

Dynamic table - "custom" headers???? need to understand this better

Binary transit

HTTP/1.1 is a text protocol, all message are sent as text over the wire wasting lots of space.

HTTP/2 uses a binary encoding of the text following

Telnet can no longer be used for debugging http/2, but Wireshark already supports it!

The HTTP/2 Connection

  • Messages : request/response
  • Streams : bi-directional channel that carries messages in both directions simultaneously
  • Frames : messages are broken up into frames for transmission, contains a (message|stream) pair id

    • Frames

    • 24 bits to mark length of payload
    • 8 bits to indicate the type of frame (PUSH, RST_STREAM, DATA, etc)
    • 8 bits for frame specific flags
    • 1 bit reserved for giggles, always 0
    • 31 bits to uniquely identify the HTTP/2 stream - clients are always odd, servers are even to prevent collision
    • The rest is payload depending on the length specified

    Frame Types

    • DATA (0x0)
    • HEADERS (0x1)
    • PRIORITY (0x2)
    • RST_STREAM (0x3)
    • SETTINGS (0x4)
    • PUSH_PROMISE (0x5)
    • PING (0x6)
    • GOAWAY (0x7)
    • WINDOW_UPDATE (0x8)
    • CONTINUATION (0x9)
